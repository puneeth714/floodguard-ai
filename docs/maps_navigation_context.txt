how to do this programatically  ?
To launch Google Maps programmatically with your exact custom route, you must pass waypoints through a standardized URL Scheme or API Link.
The Google Maps app recalculates directions dynamically when it opens. Therefore, you cannot pass a visual shape; you must pass explicit coordinate checkpoints that force the engine to use your roads.
The Standard Universal Link Structure
Use the official Google Maps Directions API URL format. This works universally across iOS, Android, web browsers, and backend code without needing specific SDK libraries. 
text
https://google.com|POINT2|POINT3&travelmode=driving
Use code with caution.
Key URL Parameters
origin: The starting address, city, or latitude,longitude coordinates.
destination: The final destination address or latitude,longitude coordinates.
waypoints: The secret to your custom path. A list of addresses or coordinates separated by a pipe character (|).
travelmode: Sets the mode. Use driving, walking, bicycling, or transit.
Implementation Examples
1. Python (Backend / Scripting)
Always use urllib.parse.quote() to safely encode your coordinates and pipe symbols for a URL.
python
import urllib.parse

# Define your route coordinates (lat,lng)
origin = "40.7128,-74.0060"        # New York
destination = "40.7589,-73.9851"   # Times Square
waypoints = [
    "40.7306,-73.9352",            # Custom detour point 1
    "40.7484,-73.9857"             # Custom detour point 2
]

# Join waypoints with a pipe character
waypoint_string = "|".join(waypoints)

# Encode parameters for the URL
params = {
    "api": "1",
    "origin": origin,
    "destination": destination,
    "waypoints": waypoint_string,
    "travelmode": "driving"
}

# Construct the final clickable navigation link
nav_url = f"https://google.com?{urllib.parse.urlencode(params)}"
print(nav_url)
Use code with caution.
2. JavaScript / Web Apps
In a browser context, use encodeURIComponent() to build the string dynamically.
javascript
const origin = "40.7128,-74.0060";
const destination = "40.7589,-73.9851";
const waypoints = ["40.7306,-73.9352", "40.7484,-73.9857"].join('|');

const mapUrl = `https://google.com{encodeURIComponent(origin)}&destination=${encodeURIComponent(destination)}&waypoints=${encodeURIComponent(waypoints)}&travelmode=driving`;

// Open directly in the user's Google Maps app or browser
window.open(mapUrl, '_blank');
Use code with caution.
3. Native Mobile Deep Linking (iOS & Android)
If you want to bypass the web browser entirely and force the native mobile app to launch instantly, use the google.navigation: intent scheme.
Android Intent:
text
google.navigation:q=latitude,longitude
Use code with caution.
(Note: The native Android navigation intent only supports a single destination query. For custom routes with multiple waypoints on mobile, the web link syntax https://google.com... is highly recommended because mobile OS platforms automatically intercept it and open it inside the native application seamlessly).
⚠️ Two Rules for Developers
Max 3 Waypoints Without an API Key: If you launch this free link format, Google Maps allows up to 3 intermediate waypoints. If your custom route needs more precision, you must use the paid Google Maps Directions API fetch service to calculate polylines.
Prefer Coordinates over Addresses: Text addresses force Google to perform geocoding on-the-fly, which can slightly alter your intended route. Using exact lat,lng strings guarantees the waypoint drops exactly on the specific lane or highway you planned.
