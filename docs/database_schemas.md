# Database Schema Specification: FloodGuard AI

This document details the Google BigQuery table schemas, data types, and indexing strategies utilized in the **FloodGuard AI** platform.

---

## 1. Table: `vulnerability_grids`
This table represents discrete spatial grids ($30\text{m} \times 30\text{m}$ cells) mapped across Bengaluru (focused on HSR Layout / Outer Ring Road corridor). It is used to calculate the localized Flood Vulnerability Index (FVI).

| Field Name | Data Type | Mode | Description |
| :--- | :--- | :--- | :--- |
| **`lat`** | `FLOAT` | `REQUIRED` | Latitude coordinate of grid center. |
| **`lng`** | `FLOAT` | `REQUIRED` | Longitude coordinate of grid center. |
| **`altitude`** | `FLOAT` | `REQUIRED` | Altitude in meters above sea level (from SRTM DEM / Google Elevation). |
| **`slope`** | `FLOAT` | `REQUIRED` | Terrain slope percentage (flat terrains $\approx 0$ are penalized). |
| **`drainage_capacity`** | `FLOAT` | `REQUIRED` | Baseline stormwater drain throughput capacity score ($0 - 100$). |
| **`fvi`** | `FLOAT` | `NULLABLE` | Calculated Flood Vulnerability Index (dynamic based on rainfall). |

---

## 2. Table: `drainage_network`
Represents the municipal stormwater drain (SWD) network coordinates, culverts, and status monitors (e.g. BBMP drainage grid).

| Field Name | Data Type | Mode | Description |
| :--- | :--- | :--- | :--- |
| **`drain_id`** | `STRING` | `REQUIRED` | Unique identifier (e.g., `HSR-D01`). |
| **`name`** | `STRING` | `REQUIRED` | Name or descriptive label of the drain segment. |
| **`lat`** | `FLOAT` | `REQUIRED` | Latitude coordinate of the drain node. |
| **`lng`** | `FLOAT` | `REQUIRED` | Longitude coordinate of the drain node. |
| **`status`** | `STRING` | `REQUIRED` | Current operational state (`"blocked"` or `"cleared"`). Used for what-if simulation tests. |

---

## 3. Table: `active_sos`
Logs real-time distress signals and multimodal alerts posted by residents.

| Field Name | Data Type | Mode | Description |
| :--- | :--- | :--- | :--- |
| **`session_id`** | `STRING` | `REQUIRED` | Session identifier linked to the resident's local storage UUID. |
| **`user_id`** | `STRING` | `REQUIRED` | Resident persona profile name (`"rajesh"`, `"radha"`, or `"anonymous"`). |
| **`lat`** | `FLOAT` | `REQUIRED` | Current GPS latitude coordinates of the distress signal. |
| **`lng`** | `FLOAT` | `REQUIRED` | Current GPS longitude coordinates of the distress signal. |
| **`detected_depth`** | `FLOAT` | `NULLABLE` | Estimated water depth in feet (derived from photo upload via Gemini Vision). |
| **`photo_url`** | `STRING` | `NULLABLE` | URL of the uploaded compound/street image. |
| **`status`** | `STRING` | `REQUIRED` | Current rescue status (`"pending"` or `"rescued"`). |
| **`timestamp`** | `TIMESTAMP` | `REQUIRED` | Time when the SOS warning was flagged. |

---

## 4. Table: `guidelines_vector`
Houses vectorized chunks of official disaster guidelines, climate action plans, and engineering manuals for the Policy & Mitigation Agent (RAG).

| Field Name | Data Type | Mode | Description |
| :--- | :--- | :--- | :--- |
| **`chunk_id`** | `STRING` | `REQUIRED` | Unique text chunk slug (e.g., `engineering_p14_a8f9`). |
| **`doc_name`** | `STRING` | `REQUIRED` | Source filename (e.g., `DPR_Guidelines_Flood_management_Works_12Apr2018.pdf`). |
| **`page_num`** | `INTEGER` | `REQUIRED` | Source PDF page number. |
| **`section_title`** | `STRING` | `NULLABLE` | Header text of the parsed PDF section. |
| **`category`** | `STRING` | `REQUIRED` | Modular classification (`"ENGINEERING"`, `"MITIGATION"`, `"CLIMATE"`, `"ADMIN"`, `"EMERGENCY"`). |
| **`content`** | `STRING` | `REQUIRED` | Plain text contents of the chunk. |
| **`embedding`** | `FLOAT` | `REPEATED` | 768-dimensional float embedding array generated by Vertex AI `text-embedding-004`. |

---

## 5. Table: `low_lying_hotspots`
Contains municipal coordinates of historically low-lying flood points in Bengaluru. Grounded in actual KML records.

| Field Name | Data Type | Mode | Description |
| :--- | :--- | :--- | :--- |
| **`name`** | `STRING` | `REQUIRED` | Location name (e.g., `Gurudarshan Layout,Near Doddabommasandra`). |
| **`lat`** | `FLOAT` | `REQUIRED` | Latitude coordinate of the flood point. |
| **`lng`** | `FLOAT` | `REQUIRED` | Longitude coordinate of the flood point. |
| **`object_id`** | `INTEGER` | `REQUIRED` | Unique database record index. |
